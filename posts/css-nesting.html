CSS Nesting is Here

Native CSS nesting in the browser is finally available in most browsers. I've been using CSS nesting for years but it's always been used with SCSS so it's nice to finally have it integrated. So, why use nesting?

  • Easier to read
  • Group styles together
  • Nesting media queries

Versions - Strict vs Relaxed

Currently there are two versions when using nesting: Strict and Relaxed. The difference being that the strict version requires the & symbol before the selector.

Nesting using Strict mode

.card {
  & .title {
    border-block-start: 2px solid blue;
  }
}

Nesting using Relaxed mode (without the nesting modifier). If the nesting selector doesn't have an &, it gets one added in front of it implicitly. 

.card {
  .title {
    border-block-start: 2px solid blue;
  }
}

Compound selectors

If you need the rule to be attached without whitespace, the nesting selector must be immediately prepended with an &. 

.parent-rule {
  &:hover {

  }
}

This is equal to .parent-rule:hover {}

Appending

The nesting selector can be appended at the end to reverse the order.

.parent-rule {
  .child-rule & {

  }
}

This is equal to .child-rule .parent-rule.

Multiple Uses

.card {
  .featured & & & {}
}

This is equal to .featured .card .card .card {}

What doesn't work?

Using BEM style concatenation won't work with the version 1 of CSS nesting. Maybe in later versions this will get added. 

.card {
  &--header {
    font-size: small;
  }
}

This won't work

Warnings

Make sure to put all parent declarations before the nested styles, not after.

.card {
  color: red; /* place parent styles above any nested elements */
  & .section {

  }
  color: blue; /* don't place here */
}
